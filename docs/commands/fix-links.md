# Fix Links Command

Automatically fixes broken anchor links in MDX files based on validation reports.

## Usage

```bash
writechoice fix links [options]
```

## Options

| Option            | Alias | Description                          | Default              |
| ----------------- | ----- | ------------------------------------ | -------------------- |
| `--report <path>` | `-r`  | Path to JSON validation report       | `links_report.json`  |
| `--quiet`         | -     | Suppress terminal output             | `false`              |
| `--verbose`       | -     | Show detailed output for each fix    | `true` (if not quiet)|

## How It Works

The fix command reads a JSON validation report generated by `writechoice check links` and automatically corrects broken anchor links in your MDX files.

### What Gets Fixed

The command fixes anchor links where:
- The target heading exists but uses a different anchor slug
- The report contains both the incorrect anchor and the correct anchor
- The link can be unambiguously located in the source file

### What Doesn't Get Fixed

- Links to non-existent pages or headings
- Links with errors (e.g., network failures, timeouts)
- Links where the target heading couldn't be determined
- External links (only internal anchor links are fixed)

### Fix Process

1. **Read Report**: Parses the JSON report to find all fixable failures
2. **Locate Links**: Finds each broken link in the source MDX file by line number
3. **Update Anchors**: Replaces the incorrect anchor with the correct one
4. **Preserve Format**: Maintains the original link format (markdown, HTML, or JSX)
5. **Write Changes**: Saves the updated file with corrected anchors

## Examples

### Basic Usage

```bash
# Fix links using default report file
writechoice fix links

# Fix links from custom report file
writechoice fix links -r custom_report.json
```

### Quiet Mode

```bash
# Suppress output (useful for CI/CD)
writechoice fix links --quiet
```

### Verbose Output

```bash
# Show detailed information for each fix
writechoice fix links --verbose
```

## Example Fix

**Before** (incorrect anchor):
```markdown
[Installation Guide](./installation#setup)
```

**After** (corrected anchor):
```markdown
[Installation Guide](./installation#setup-process)
```

The fix command identifies that the heading "Setup Process" generates the anchor `#setup-process`, not `#setup`, and updates the link accordingly.

## Report Requirements

The fix command **requires a JSON report file**. If you try to use a Markdown report, you'll get an error:

```
✗ Error: The fix command requires a JSON report file.

The markdown (.md) report is for human readability only.
Please use the JSON report instead: links_report.json
```

This is because:
- JSON contains structured data needed for fixing (line numbers, anchors, file paths)
- Markdown is a presentation format for humans, not machines

## Recommended Workflow

1. **Run validation**:
   ```bash
   writechoice check links https://docs.example.com
   ```
   This generates `links_report.json` and `links_report.md`

2. **Review the Markdown report**:
   ```bash
   cat links_report.md
   ```
   Check which links are broken and understand the issues

3. **Apply fixes**:
   ```bash
   writechoice fix links
   ```
   This reads `links_report.json` and fixes broken anchor links

4. **Review changes**:
   ```bash
   git diff
   ```
   Verify the changes look correct before committing

5. **Re-validate**:
   ```bash
   writechoice check links https://docs.example.com
   ```
   Confirm all issues are resolved

## Output Format

### Success Output

```
🔧 Link Fixer

Reading report: links_report.json
Fixed path/to/file1.mdx:42
  Old: ./installation#setup
  New: ./installation#setup-process
Fixed path/to/file2.mdx:15
  Old: /api/reference#authentication
  New: /api/reference#authentication-methods

✓ Fixed 2 link(s) in 2 file(s):
  path/to/file1.mdx: 1 fix(es)
  path/to/file2.mdx: 1 fix(es)

⚠️  Run validation again to verify the fixes:
  writechoice check links <baseUrl>
```

### No Fixes Available

```
🔧 Link Fixer

Reading report: links_report.json

⚠️  No fixable issues found in report.
```

This can happen if:
- All links are already correct
- The failures can't be automatically fixed (e.g., missing pages)
- The report contains only errors, not failures

## Safety Features

### Non-Destructive

The fix command:
- Only modifies files that have fixable issues
- Preserves the original link format (markdown, HTML, JSX)
- Maintains indentation and formatting
- Only changes the anchor portion of links, not the path

### Verification

After running fixes:
1. Use `git diff` to review all changes
2. Run validation again to confirm fixes worked
3. Test your documentation locally before deploying

### Reversibility

All changes can be reverted using git:
```bash
# Undo all fixes
git checkout .

# Undo changes to specific file
git checkout path/to/file.mdx
```

## Troubleshooting

### Report File Not Found

```
✗ Error: Report file not found: links_report.json

⚠️  Make sure to run the validation command first:
  writechoice check links <baseUrl>
```

**Solution**: Run `writechoice check links` to generate the report first.

### Markdown Report Provided

```
✗ Error: The fix command requires a JSON report file.

The markdown (.md) report is for human readability only.
Please use the JSON report instead: links_report.json
```

**Solution**: Use the `.json` file instead of the `.md` file.

### Link Not Found on Line

```
Warning: Could not find href './installation#setup' on line 42 in path/to/file.mdx
```

**Possible causes**:
- The file was modified after the report was generated
- The link spans multiple lines
- The link format changed

**Solution**: Re-run validation to generate a fresh report.

## Integration with CI/CD

### GitHub Actions Example

```yaml
name: Fix Documentation Links
on:
  workflow_dispatch:

jobs:
  fix-links:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm install -g writechoice-mint-cli

      - name: Validate links
        run: writechoice check links https://docs.example.com

      - name: Fix links
        run: writechoice fix links

      - name: Create Pull Request
        uses: peter-evans/create-pull-request@v5
        with:
          title: 'Fix broken documentation links'
          commit-message: 'fix: update broken anchor links'
          branch: fix/documentation-links
```
